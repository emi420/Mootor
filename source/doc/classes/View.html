View

Defined in: js/view.js:1
Module: View

Summary

The View class handles each view of the application. A list of views is specified in the applications options and the files are loaded from the "views" folder. Each view has a viewName.js, viewName.html and viewName.css files.

Summary

Unsets event handlers for the view Possible values for event: load, beforeLoad, unload, beforeUnload, init

Parameters:

  • event String

    Defines in which event we want to unset handlers

  • [callback] Function optional

    If this parameter is specified, only that function is removed. Otherwise all callbacks for this event are removed.

on

Defined in js/view.js:266

Syntax

on

(
  • event
  • callback
)

Summary

Sets an event handler for the view Possible values for event: load, beforeLoad, unload, beforeUnload, init

Parameters:

  • event String

    Defines in which event the handler will be called

  • callback Function

    The function to be called when the event is fired.

Returns:

View

Example:

// Simple example

m.app.route("^#index$", app.view("index"));

m.app.view("index").on("load", function(self) {
   console.log("Index view is loaded."); 
});

// With parameters

m.app.route("^#product/(.*)", app.view("product"));

m.app.view("product").on("load", function(self) {
   console.log("Product Id: " + self.params[0];
});

title

Defined in js/view.js:242

Syntax

title

(
  • [title]
)

Summary

Title is the friendly name for the current view. When called without parameters, returns a string containing the title. When called with parameters, sets the title.

Parameters:

  • [title] String optional

    New title for this view.

Returns:

string

Example:

// Get title
m.app.view("index").title();

// Get title
m.app.view("index").title("My title");
